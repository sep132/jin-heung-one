##  html에 class 넣을 땐 className

* 왜냐?
    * App.js에 짜고 있는건 html이 아니라 JSX라고 부르는 이상한 언어라서임
    * 원래 리액트환경에서 <div>하나 만들고 싶으면 자바스크립트로
```javascript
    React.createElement('div', null) 

    // 이딴 식으로 어렵게 코드짜야
```

#### 근데 이러면 코딩이 더 어려워지기때문에 JSX라는 언어를 사용함
* jsx는 html과 사용방식은 비슷함
* 근데 jsx는 일종의 자바스크립트라서 자바스크립트에서 사용하는 예약어인 class를 막 사용하면 안됨

* ## 그래서 html의 class =" " 를 사용하고싶다면,
* # className이라고 써야됨
* ### jsx의 기본적인 첫번째 문법임

## 변수를 html에 꽂아넣을 때는 {중괄호}

## state 문법을 쓰는 이유
* ### state는 변동사항이 생기면 state쓰는 html도 자동으로 재렌더링
* #### state는 상품명, 글제목, 가격 이런것 처럼 자주 변할 것 같은 데이터들을 저장하는게 좋은 관습입니다.
* ## state 요약
    * 자주변경될 것 같은 데이터들은 state에 저장했다가 html에 {데이터바인딩} 해놓으십시오

    * 변경할 일이 없는 데이터들

    * 굳이 html에 표기가 필요없는 데이터들은 그냥 변수에 저장해도 됩니다.
